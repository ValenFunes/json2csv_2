## Project purpose

`json2csv` converts JSON files to CSV format. It is a Python CLI tool and importable library, following the same conventions established by the `csv2json` class reference project.

## Project layout

```
json2csv/
├── src/
│   └── json2csv/
│       ├── __init__.py       # public re-exports + __version__
│       ├── converter.py      # core logic (flatten, load, normalize, write, convert)
│       └── cli.py            # argparse CLI entry point
├── tests/
│   └── test_converter.py     # pytest unit tests (≥ 85 % coverage)
├── docs/                     # auto-generated by pdoc
├── .github/
│   └── workflows/
│       └── ci.yml            # full CI/CD pipeline
├── pyproject.toml            # build + all tool configuration
├── requirements.txt          # dev/CI dependencies
├── VERSION                   # e.g. 0.1.0
├── BUILD                     # monotonic integer, bumped by CI
├── LICENSE                   # MIT
├── README.md
├── CHANGELOG.md
└── CONTEXT.md                # ← this file
```

## Architecture decisions

| Decision | Rationale |
|---|---|
| `src/` layout | Prevents accidental import of un-installed package during tests |
| Typed exceptions hierarchy | Callers can catch specific vs generic errors |
| `flatten_dict` recursion | Handles arbitrarily deep nesting |
| stdlib only (no pandas) | Zero runtime dependencies — simpler deployment |
| `pdoc` for docs | Minimal config, auto-generates from docstrings |

## Conventions

- All public functions have Google-style docstrings (PEP 257 + ruff D).
- Type hints everywhere (`from __future__ import annotations`).
- Line length: 100 characters.
- Commits follow Conventional Commits (`feat:`, `fix:`, `test:`, `ci:`, `docs:`).
- Branch naming: `username/ISSUE-description`.

## CI pipeline summary

1. `ruff check` — lint + PEP 8 rules
2. `black --check` — formatting
3. `pydocstyle` — PEP 257 docstrings
4. `mypy` — static type checking (strict)
5. `pyright` — additional type checking
6. `bandit` — security scan (no TruffleHog)
7. `pytest --cov` — unit tests, ≥ 85 % coverage required
8. `pdoc` — generate and publish docs (on successful PR merge)
9. Bump `BUILD` number

## Known limitations / future work

- Nested lists of objects are serialised as JSON strings (not expanded).
- No streaming support for very large files.
- No schema validation of input JSON.
